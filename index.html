Conference Article: Is the Decentralisation of Web 3.0 Achievable in its Current State?

Abstract

Web 3.0 was developed as a decentralised web network and guarantees the decentralisation of user data and the clearing of power structures inherent in Web 2.0 (Alabdulwahhab, 2018). This article evaluates the degree of realisable decentralisation within the blockchain-based Web 3.0 platform, as it explores its usability, regulated token operations, congestion, and environmental and compliance-related issues (Reiff, 2024; Earthjustice, 2022). There is promise of a user-friendly internet, but technical and operational problems still need to improve the full adoption of Web 3.0. The article delineates the advancement and the restrictions in the actualisation of decentralisation, thus, concluding that the substantial gains in technology, regulation, and users' adoption of the technology are prerequisites for the Web 3.0 vision to come to fruition, and as a result, technology, law, and user participation must impute.

Introduction

The internet’s evolution has drastically changed how humans interact with each other and the world around us. With this, society has shifted, becoming more data-driven as our data is ever more valuable and out of our control (Swan, 2015). Web 3.0 now introduces decentralisation as its defining feature. Coined by Gavin Wood in 2014, Web 3.0 envisions a “trustless” Internet where users control their data without central oversight (Alabdulwahhab, 2018). However, in a world dominated by major corporations such as Meta and Google, this article questions the feasibility of Web 3.0’s decentralisation in its current form (Kim, 2019). Through an exploration of blockchain technology, token economics, and systemic challenges, the article aims to determine if Web 3.0 can indeed achieve decentralisation or if it remains an idealistic vision (Swan, 2015).

The Evolution of the Web

Understanding Web 3.0’s decentralisation aims requires examining previous web incarnations and their effects on data control.

Web 1.0

Web 1.0 was the first stage of the Internet. It was mainly “read-only,” featuring static pages with limited user interaction. Built on basic technologies like HTML and HTTP, Web 1.0 established core protocols for data retrieval and information sharing without centralised data management (Geroni, 2021).

Web 2.0

Web 2.0 transformed the internet by enabling user interaction, content creation, and social media growth. Platforms like Facebook and YouTube allow companies to collect and monetise large amounts of user data (Reiff, 2024). Companies such as Meta and Amazon now control much of the infrastructure behind Web 2.0, making today’s internet highly centralised (Swan, 2015).

The Promise of Web 3.0 Decentralisation

By utilising blockchain technology and decentralised protocols, Web 3.0 seeks to remove centralised power structures that are used in Web 2.0. The main goal is to allow users to transact directly without middlemen, returning control of data to the users (Alabdulwahhab, 2018). Essential Web 3.0 innovations such as peer-to-peer transactions and decentralised applications (DApps) aim to replace central servers with a network of computers managed by the users themselves (Marlinspike, 2022).
